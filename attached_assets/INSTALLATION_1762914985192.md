# Installation Guide for TaptPay Landing Page

## Quick Start

### 1. Install Dependencies

```bash
npm install lucide-react motion react
```

Or with yarn:
```bash
yarn add lucide-react motion react
```

### 2. Copy Files

Copy the following files to your project:

```
your-project/
├── components/
│   ├── LandingPage.tsx        (copy from package)
│   └── HowItWorksSection.tsx  (copy from package)
```

### 3. Add Font (Outfit from Google Fonts)

Add this to your `index.html` or layout:

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Outfit:wght@300;400;500;600;700&display=swap" rel="stylesheet">
```

Or in your CSS:

```css
@import url('https://fonts.googleapis.com/css2?family=Outfit:wght@300;400;500;600;700&display=swap');
```

### 4. Replace Image Imports

The landing page uses Figma assets. Replace these with your own images:

```tsx
// Find and replace these imports in LandingPage.tsx:
import logo from 'figma:asset/...';                  // Your logo
import activeTransactionsImage from 'figma:asset/...'; // Dashboard phone mockup
import paymentStonesImage from 'figma:asset/...';      // Payment stones image
import customerPhoneImage from 'figma:asset/...';      // Customer payment mockup
import terminalReadyImage from 'figma:asset/...';      // Terminal feature 1
import terminalSplitBillImage from 'figma:asset/...';  // Terminal feature 2
import terminalShareImage from 'figma:asset/...';      // Terminal feature 3
import terminalNfcImage from 'figma:asset/...';        // Terminal feature 4

// Replace with:
import logo from './assets/logo.png';
import activeTransactionsImage from './assets/dashboard.png';
// ... etc
```

### 5. Handle ImageWithFallback Component

The landing page uses `ImageWithFallback` component. You have two options:

**Option A:** Replace with standard img tags

Find all instances of:
```tsx
<ImageWithFallback src={...} alt="..." style={...} />
```

Replace with:
```tsx
<img src={...} alt="..." style={...} />
```

**Option B:** Create a simple ImageWithFallback component

Create `components/figma/ImageWithFallback.tsx`:

```tsx
interface ImageWithFallbackProps {
  src: string;
  alt: string;
  style?: React.CSSProperties;
  className?: string;
}

export function ImageWithFallback({ src, alt, style, className }: ImageWithFallbackProps) {
  return <img src={src} alt={alt} style={style} className={className} />;
}
```

### 6. Update Imports

In `LandingPage.tsx`, update the import path:

```tsx
// If you chose Option B above:
import { ImageWithFallback } from './figma/ImageWithFallback';

// If you chose Option A, remove this import completely
```

### 7. Use the Component

```tsx
import { LandingPage } from './components/LandingPage';

function App() {
  const handleGetStarted = () => {
    // Handle navigation to signup/login
    window.location.href = '/signup';
  };

  return (
    <div>
      <LandingPage onGetStarted={handleGetStarted} />
    </div>
  );
}

export default App;
```

## Tailwind CSS Configuration

This landing page uses Tailwind CSS v4.0. If you're using an older version, you may need to:

1. **Ensure Tailwind is installed:**
```bash
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init
```

2. **Configure Tailwind** (tailwind.config.js):
```js
module.exports = {
  content: [
    "./index.html",
    "./src/**/*.{js,ts,jsx,tsx}",
  ],
  theme: {
    extend: {
      fontFamily: {
        outfit: ['Outfit', 'sans-serif'],
      },
    },
  },
  plugins: [],
}
```

3. **Add to your main CSS file:**
```css
@tailwind base;
@tailwind components;
@tailwind utilities;
```

## Optional: Global CSS Customization

If you want to match the original design system exactly, add these to your `globals.css`:

```css
@layer base {
  body {
    font-family: 'Outfit', sans-serif;
  }
}
```

## Troubleshooting

### Issue: Motion/Framer Motion not working

**Solution:** Make sure you're importing from `motion/react`:
```tsx
import { motion, AnimatePresence } from 'motion/react';
```

NOT:
```tsx
import { motion } from 'framer-motion'; // ❌ Old syntax
```

### Issue: Images not loading

**Solution:** 
1. Verify image paths are correct
2. Ensure images are in your `public` or `assets` folder
3. Use relative paths like `./assets/logo.png` or absolute paths like `/images/logo.png`

### Issue: Fonts not displaying correctly

**Solution:**
1. Check that Google Fonts link is in your HTML
2. Verify the font-family is applied: `font-['Outfit']` in className
3. Clear browser cache

### Issue: Tailwind classes not working

**Solution:**
1. Ensure Tailwind is properly configured
2. Check that file paths in tailwind.config.js include your components
3. Restart your dev server

### Issue: Carousel not auto-rotating

**Solution:** Check that the useEffect hook is properly set up with the nextSlide dependency
